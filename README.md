# AutoPrototype

**Project Goal:** To build a fully autonomous agentic workflow that transforms a natural language idea (e.g., "Build a water tracker") into a functional, multi-file software prototype using FastAPI and React.

## Current Project Status: Week 3 (The Builders)
We implemented the core "Factory" logic. The system currently generates an architecture plan, writes the backend/frontend code, and organizes it into a directory structure.

## Technical Architecture: How it Works

AutoPrototype is a **State Machine** built with **LangGraph**. It mimics a real-world software team by passing context through a shared "Whiteboard"

```bash
auto-prototype/
├── agents/               # AI Agent logic
│   ├── dev_agents.py     # Backend & Frontend Developer nodes + Smart Saver
│   └── pm_agent.py       # Product Manager node
├── core/                 # LangGraph Orchestration
│   ├── graph.py          # State Machine & Edge definitions
│   └── state.py          # Shared "Whiteboard" (AutoPrototypeState)
├── sandbox/              # The Execution Engine (Week 1 Setup)
│   ├── Dockerfile        # Isolated environment for testing AI code
│   └── executor.py       # Python script to run code inside Docker
├── output_prototype/     # THE GENERATED PRODUCT (AI-Built)
│   ├── architecture_plan.md
│   ├── backend/          # FastAPI App
│   └── frontend/         # React App
├── .env                  # API Keys (OpenAI)
├── .gitignore            # Ignore .venv, __pycache__, and .env
├── main.py               # Entry point to run the factory
├── README.md             
└── requirements.txt      # Project dependencies
```

### 1. The Shared State (`core/state.py`)
All agents communicate via the `AutoPrototypeState`. This is a dictionary that persists throughout the run, holding:
* `architecture_plan`: The single source of truth for requirements.
* `backend_code` & `frontend_code`: The raw strings generated by the developer agents.

### 2. The Agent Nodes (`agents/`)
* **Product Manager Agent**: High-level strategist. It defines the tech stack and file structure.
* **Developer Agents**: Frontend and Backend coders (FastAPI & React). They follow a **Strict Formatting Contract** using `### path/to/file` headers.
* **Smart File Saver**: Custom utility. It uses **Regular Expressions (Regex)** to organzie the AI's response and create the directories and files on your disk.

### 3. The Sandbox (`sandbox/`)
This is our "Safety Net." We use a Docker Container to execute the AI-generated code. This ensures that:

* We don't crash our local machines if the AI writes a "loop of death."

* We have a clean, reproducible environment for testing.

* The Debugger (Week 4) can capture real error logs from a simulated server.

### 3. The Workflow Graph (`core/graph.py`)
The flow is orchestrated as a directed relay:
**Idea** → **PM** → **Backend Dev** → **Frontend Dev** → **File Saver** → **Success**.

## Setup Instructions

Follow these steps to get the factory running on your local machine:

### 1. Prerequisites
* **Python 3.12** (WSL/Linux preferred).
* **OpenAI API Key** (for GPT-4o).

### 2. Installation
```bash
# Create and activate virtual environment
python3 -m venv .venv
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```
### 3. Environment Variables
```bash
# Create a .env file in the root:
OPENAI_API_KEY=your_actual_key_here
```
### 4. Running the Workflow
```bash
python3 main.py
```
After the run completes, check the output_prototype/ folder. You will find:

* architecture_plan.md
* backend/ (Full FastAPI structure)
* frontend/ (Full React structure)
