Trials and Tribulations:
- Initially used GPT4-o; however, we changed to claude-3-5-sonnet for the agents since the model seems to be better at reasoning and coding,
However, it is significantly more expensive, and the feedback loop was changed so GPT4-o might work just as well (will possibly switch back in the future
after some more testing)
- The debugger agent was originally evaluating the code statically; it just checked if the code written looked correct. We changed it to evaluate dynamically
by running the sandbox Docker after every iteration and giving the Docker logs to the Debugger agent.
- We added a lot of specific, hardcoded rules; however, it caused the agents to perform worse, so they were removed and more generalized
- Backend agent got over ambitious and started to write the entire codebase in one go. Which caused the frontend folder to be inside the backend folder. Causing the executor node to fail. 
- Backend agent doesn't know their backend code is going to be executed by the exectuor node so it has to guess the root package of the project. 
- In autonmous software engineering, the execution sandbox must faithfully mirror the agent's intentions. Can't hardcore enviroment variables configurations that ai cannot access.
